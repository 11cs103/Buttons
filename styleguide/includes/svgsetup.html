Inline SVG Notes

Using inline SVG requires a bit of setup–but we've got covered with the live example you're viewing right now and links galore! In fact, the svg-builder directory in this very project, contains a complete inline SVG build workflow. It requires you're understanding of things like Grunt and a bit of comfort with the command line. You may want to first do a bit of reading to get some background on how this all works. css-tricks.com definitely takes the prize when it comes to getting up to speed with using inline SVG. Here are some hand-picked posts on the topic:

  • Icon Font vs. SVG–this is the one that convinced me it was time to try using inline SVG in a production system.
  • Using SVG–just a great overview on SVG in use.
  • ROBS POST TBD–this article lists all gotchas and workarounds I came up against (this WILL save you time if you're doing this “for reals”)
  • SVG Use External Source–information on caching SVG by using an external defs file.
  • Swapping Icons–some very practical tips on how to, well, swap icons.
  • All the css-tricks articles tagged "svg"–since Chris has become a bit of curator on the topic, this is useful too!

Ok…ok, the tl;dr is to just go to the svg-builder directory and type 

npm install && grunt
And then go ahead and have a look at the Gruntfile.js
